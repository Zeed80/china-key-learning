# Learning Design Notes

Цель страницы ключа: быстрое распознавание формы, привязка к смысловой зоне и различение похожих начертаний без перегруза длинной статьей.

## Principles

- Retrieval practice: ключи должны быстро перелистываться и затем проверяться тренировкой, потому что активное извлечение лучше пассивного перечитывания для долгого удержания.
- Spacing: сильные ключи не исчезают из ротации, но появляются реже; слабые возвращаются чаще.
- Desirable difficulty: рядом показываются похожие формы, чтобы пользователь учился различать близкие визуальные признаки.
- Cognitive load: первый экран карточки держит один главный фокус: глиф, краткий смысл, отличие и 1-3 примера. Длинные разборы ограничены компактными прокручиваемыми блоками.
- Dual coding: форма, короткое текстовое правило, пример-иероглиф и предложение показываются вместе.

## Applied UI Rules

- ArrowLeft/ArrowRight листают ключи без перехода в поиск.
- Каталог остается слева для прямого перехода к любому из 214 ключей.
- Основной глиф и варианты формы закреплены визуально крупнее текста.
- Примеры содержат иероглиф, pinyin, перевод, разбор компонентов и короткое предложение.
- Похожие формы показываются парой символов; отличающие признаки подсвечиваются.
